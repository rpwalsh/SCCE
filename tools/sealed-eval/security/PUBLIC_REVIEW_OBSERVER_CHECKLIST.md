# Public review observer checklist

The observer verifies procedure, not secret formulas.

- [ ] Source snapshot hash visible.
- [ ] Build performed or build hash verified.
- [ ] Network condition visible.
- [ ] Corpus/question seal verified.
- [ ] Questions were not supplied by the system owner unless that role overlap is disclosed.
- [ ] System and ablation order randomized.
- [ ] Raw outputs closed and hashed.
- [ ] No answer editing occurred.
- [ ] Blinded judging completed before unblinding.
- [ ] All failures and deviations appear in the package.
- [ ] Reported numbers resolve to raw artifacts.
- [ ] Synthetic rehearsals are labeled as rehearsals and public-review status remains `NOT_EXECUTED` until a custodian-sealed evaluation closes.
- [ ] Any VS Code parity claim resolves to a packaged extension-host run, not only unit tests or source inspection.
- [ ] Any coding-agent claim resolves to a sealed coding adapter run and not only reviewed application of a caller-supplied patch plan.
